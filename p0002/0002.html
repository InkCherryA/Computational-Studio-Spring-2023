XR Interaction Toolkit: Part I
Following the section 2 of the course VR Development Fundamentals With Oculus Quest 2 And Unity on Udemy, functionality of the XR Interaction Toolkit is tested. Using the imported Harry Potter model asset, custom hand asset, and XR Interaction Toolkit sample asset, simple VR actions are implemented, such as ray grabbing, direct grabbing, joystick moving, joystick turning.

Using the hand controller script, we are able to get float values from the VR controller device to transition smoothly between multiple poses of the hand model. This is done by implementing an animation blend tree, where four different poses are mapped onto a 2-dimensional space. The float values of the two buttons, grip and trigger, will translate as the x and y coordinate in this space.

Using the scripts that are imported from the XR Interaction Toolkit sample asset, a locomotion system can be set up on the XR Origin (player) game object, allowing the player to move, continuous or teleport, and turn, continuous or snap, using the joysticks on the controller devices. An interaction system is also provided by the asset which allows the player to grab objects that are deemed interactable. The tutorial also explained the differences between the movement types of an interactable, instantaneous, kinematic, and velocity tracking. We are also able to control the haptic behavior of the controller through these scripts.

XR Interaction Toolkit: Part II
To implement the knowledge gained from this tutorial to our own game design, we had to make some changes to the hand model. We downloaded a 3d model of a mechanical hand from Turbosquid, which is then rigged in Blender. We used the Damped Track and Limit Rotation bone constraints to make the movement of the fingers mechanically accurate. The meshes, armature, along with two poses, relaxing and grabbing, are imported into Unity through an intermediate FBX file.

Because we currently have only two poses, we are able to achieve the smooth hand animation using a 1-dimensional instead of 2-dimensional blend tree, with the grip button value as a parameter. We minimally modified the Hand Controller Script provided by the turorial to get the required value from the device.
