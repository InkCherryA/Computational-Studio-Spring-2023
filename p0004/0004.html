Rendering in Unity: Part II
We modeled the rough geometry, wall and floor, of the room in Blender, based on the selected concept map that was generated by Midjourney. We then imported the mesh into Unity through an FBX file.

We configured the scene and environment tabs in the lighting window. We enabled baked global illumination, tweaked the lightmap resolution, and disabled all existing environment lighting to isolate effects of the scene lighting.

We also tested the procedual texture generation in both Blender and Unity. In Blender, this can be achieved simply through the shader nodes that is avaiable, while in Unity, we must first replace the standard render pipeline with either universal render pipeline (URP) or high-definition render pipeline (HDRP). After that, custom shader graphs can be applied to materials. For the wall, we are trying to create metallic panels that have vertical grains of reflectivity. For the floor, we want simple tiles that vary in color.

We created several light sources using geometries with emissive materials. The indirect lighting information are then baked onto the wall and floor, generating lightmaps corresponding to the unwrapped UV map exported from Blender.

We compared results of the standard and universal render pipeline. We were not able to use procedurally generated shader materials in the standard RP. To add a bloom effect to the scene, we used the post-process volume, and the global volume for the two RPs respectively. In order for the reflective wall material to show up, we had to add a reflection probe to the URP version.

Later, we decided to use the procedually generated texture from Blender. We baked the textures, diffuse, normal, and roughness, into images, and applied them onto the game objects using a custom shader graph.
